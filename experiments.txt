Model Experiments Log
=====================

This file documents all experiments run to improve the NCAA basketball
prediction model. The baseline model uses season-level KenPom stats as
features and predicts game margins via regression (win probability is
derived from predicted margin using a logistic function).

Training set: ~91k games from 2010-2026 seasons. Pre-2010 data is in the
DB but excluded from training because it lacks KenPom stats.


Baseline (as of 2026-03-09)
----------------------------
Model: GradientBoostingRegressor (200 trees, max_depth=4)
Features: 26 stats (all available team_stats except games_played)
Margin MAE: 7.674
Win accuracy: 80.0%
Top feature: adj_efficiency_margin at 89% importance


1. Feature Selection (2026-03-09)
----------------------------------
Experiment: Compare top 8 features vs all 26.
Result: Top 8 = best accuracy (MAE 7.661, 80.03% win acc). All 26 slightly
worse. Extra features add noise without improving predictions.

Locked-in top 8: adj_efficiency_margin, pythag, avg_margin, win_pct,
luck, sos, sos_offense, + neutral_site flag.


2. Recent Form — Last 10 Games (2026-03-09)
---------------------------------------------
Experiment: Add rolling stats from each team's last 10 games.
Result: 0.27% feature importance. No improvement. KenPom's ratings
already incorporate recent performance implicitly.


3. Roster Features (2026-03-09)
--------------------------------
Experiment: Add height, experience, continuity, and bench stats from
KenPom's height endpoint.
Result: <0.25% importance each. Completely redundant with KenPom
efficiency ratings, which already reflect the impact of roster quality.


4. Removing adj_efficiency_margin (2026-03-09)
------------------------------------------------
Experiment: Train the model without its dominant feature.
Result: MAE 7.685 vs 7.674 (barely worse). pythag takes over at 79%
importance. The model is fundamentally just learning "better KenPom
rating wins" — it doesn't matter which KenPom-derived metric it uses.


5. Point-in-Time Ratings (2026-03-09)
---------------------------------------
Experiment: Use KenPom archive ratings from the date of each game instead
of end-of-season values. Built team_ratings_by_date table with ~2.87M
historical ratings across 1,877 dates.
Result: Worse — MAE 8.23, 77.5% win acc vs 7.66, 80.0%. Early-season
PIT ratings are noisy and unreliable. End-of-season values are actually
a better estimate of true team quality, even for early-season games.
Infrastructure (build_training_data_pit) retained but not used.


6. 30-Day AdjEM Delta / Momentum (2026-03-09)
------------------------------------------------
Experiment: Add the change in adj_efficiency_margin over the last 30 days
as a momentum signal.
Result: 0.29% importance. Trend information is redundant with the
season-level efficiency values.


7. AdjOE + AdjDE Split (2026-03-10)
--------------------------------------
Experiment: Replace adj_efficiency_margin (AdjOE - AdjDE) with separate
adj_offensive_efficiency and adj_defensive_efficiency features. Hypothesis:
the model could learn that matchups between good offense vs good defense
play differently than similar overall margins.
Result: No improvement. AdjOE at 0.7%, AdjDE at 0.5% importance. pythag
takes over at 80%. MAE 7.67 vs 7.65. The offense/defense distinction
doesn't help — team quality is effectively a single dimension.


8. Rest Days (2026-03-10)
---------------------------
Experiment: Compute days since each team's last game from game dates in
the DB. Feature = home_rest_days - away_rest_days, capped at 7.
Result: 0.10% importance. Identical MAE. KenPom's efficiency ratings
already account for scheduling effects indirectly (teams on tough
schedules with short rest tend to have worse efficiency numbers).


9. MLP vs GBM (2026-03-10)
-----------------------------
Experiment: Replace GradientBoostingRegressor with MLPRegressor (sklearn).
Tested multiple configurations:

  Model               | Features | MAE   | Win Acc
  GBM                 | 8        | 7.652 | 80.05%
  GBM                 | 25       | 7.663 | 79.98%
  MLP (64-32)         | 8        | 7.605 | 80.25%
  MLP (64-32)         | 25       | 7.616 | 80.18%
  MLP (128-64-32)     | 25       | 7.625 | 80.11%

Result: MLP (64-32) with 8 features is the best configuration. Slightly
better accuracy than GBM and dramatically faster training (~10 sec vs
~3.4 min). The 8-feature set remains best regardless of model type.
Adopted MLP as the production model.


10. Opponent Adjust — Barttorvik/Miya (2026-03-10)
-----------------------------------------------------
Experiment: Import "opponent_adjust" from Barttorvik Power Ratings (Miya).
This metric measures how well a team performs above/below expectation
against strong vs weak opponents ("playing up/down to competition").
Data available 2010-2026 from CSV downloads.
Result: 0.46% importance. MAE 7.60 vs 7.61 (within noise). The signal
is already captured by the combination of efficiency + SOS + luck.
Importer code retained for potential future use with other Miya stats.


11. Home Rank — Barttorvik/Miya (2026-03-10)
-----------------------------------------------
Experiment: Import "home_rank" from Barttorvik Power Ratings. This ranks
how much better each team performs at home versus on the road. A team
ranked higher has a much larger home/away performance split. Hypothesis:
this captures team-specific home court advantage beyond the binary
neutral_site flag.
Data available 2010-2026 from same Miya CSV files.
Result: 0.17% importance. MAE identical to baseline. The neutral_site
flag already captures what matters about venue effects.


12. 3pt Shooting Reliance (2026-03-10)
-----------------------------------------
Experiment: Import F3GRate (3pt attempt rate) and OppF3GRate (opponent 3pt
attempt rate) from KenPom misc-stats endpoint. Hypothesis: teams that rely
heavily on 3pt shooting have more variance in scoring, which could predict
upsets. Ideally we'd use game-level 3pt variance, but that would require
~92k individual ESPN box score API calls.
Result: 0.16-0.18% importance. MAE 7.63 vs 7.62 (within noise). 3pt
reliance is already captured by the efficiency ratings.


13. Travel Distance (2026-03-10)
----------------------------------
Experiment: Geocode arena locations (city/state from KenPom teams endpoint)
using Nominatim. Compute haversine distance between home and away team arenas
for each game. 308 unique locations geocoded, 218/365 teams matched to DB IDs.
Result: 0.18-0.22% importance. MAE identical to baseline. Travel distance
has no meaningful effect on game outcomes beyond what's already captured
by team quality ratings.


14. XGBoost / LightGBM / Stacking Ensemble (2026-03-10)
---------------------------------------------------------
Experiment: Compare modern gradient boosting (XGBoost, LightGBM) against the
MLP and sklearn GBM. Also tested stacking ensembles combining all three via
simple averaging, weighted combinations, and a linear meta-learner.

  Model               | MAE (OOF) | Win Acc (OOF) | Time
  MLP (64-32)         | 7.615     | 80.14%        | 32s
  XGBoost (tuned)     | 7.665     | 80.08%        | 5.5s
  LightGBM (tuned)    | 7.654     | 80.11%        | 10.5s
  sklearn GBM         | 7.657     | 80.18%        | 216s
  Meta-Linear stack   | 7.614     | 80.15%        | —

Result: All models converge to the same ~80.1% / 7.61 MAE when evaluated
with out-of-fold predictions. The meta-learner assigns 81% weight to MLP,
17% to LightGBM, and 1% to XGBoost — they make nearly identical predictions.
XGBoost appeared better at 80.87% win accuracy on training data, but that was
overfitting. The accuracy ceiling is in the data, not the model architecture.
MLP retained as production model (best MAE, reasonable speed).


15. Contextual Features: Season Month, Games Played, Conference (2026-03-10)
------------------------------------------------------------------------------
Experiment: Add three non-stat contextual features:
- season_month: calendar month mapped to season progression (Nov=1..Apr=6)
- avg_games_played: average games played by both teams at game time
- same_conference: 1 if teams are in the same conference, 0 otherwise

Result: Small but consistent improvement. MAE 7.587 vs 7.612 baseline
(best improvement of any feature tested). Feature importances: same_conference
0.69%, avg_games_played 0.64%, season_month 0.39%. All three adopted into
the production model — the first new features added since the original top 8.


16. Tournament Seeds (2026-03-10)
------------------------------------
Experiment: Import historical NCAA tournament seeds from Kaggle dataset
(MNCAATourneySeeds.csv, 2,626 entries from 1985-2025). Added diff_seed
(home seed - away seed) and is_tourney flag. Tested on tournament games
only (1,001 games) and on full training set with seed=0 for regular season.
Result: 0.19% importance. No improvement. Seeds are highly correlated with
KenPom ratings — a 1-seed has a great AdjEM, a 16-seed has a terrible one.
The seed number adds no information beyond what efficiency margin captures.


17. Coach Tenure (2026-03-10)
-------------------------------
Experiment: Import coach history from Kaggle (MTeamCoaches.csv, 13,900
entries). Computed consecutive years each coach has been at their current
school. Features: diff_coach_tenure (home - away), home_new_coach flag,
away_new_coach flag.
Result: 0.17% importance for tenure diff, 0.10% for new coach flags.
MAE slightly worse (7.62 vs 7.59). A coach's impact is already reflected
in the team's KenPom ratings.


18. Point-in-Time Consensus Rank + PIT AdjEM (2026-03-11)
-----------------------------------------------------------
Experiment: Add two point-in-time (PIT) features looked up by game date from
team_ratings_by_date, in addition to existing season-level stats:
- consensus_rank: Median ordinal rank across ~60 ranking systems (Massey
  composite). Historical data from Kaggle MMasseyOrdinals.csv (156k weekly
  rankings, 2003-2026). Current data from masseyratings.com CSV export.
- pit_adj_efficiency_margin: KenPom AdjEM on the date of each game, from
  KenPom archive API (556k daily ratings, 2010-2026).

These differ from experiment #5 (PIT ratings alone) because they supplement
rather than replace the season-level features. The season-level stats capture
true team quality; the PIT features add "how is this team ranked right now"
context, which matters for early-season games and teams on hot/cold streaks.

Result: Significant improvement — the best single-experiment gain recorded.
  Metric         | Before  | After   | Change
  Win accuracy   | 80.3%   | 81.7%   | +1.4%
  Margin MAE     | 7.59    | 7.39    | -0.20

Feature importances:
  diff_consensus_rank:          10.1%
  diff_pit_adj_efficiency_margin: 8.5%

Unlike experiment #5, these PIT features are not redundant with season-level
stats because they provide complementary information: consensus_rank aggregates
60+ ranking systems (not just KenPom), and PIT AdjEM captures within-season
trajectory that end-of-season values smooth out.

Both features adopted into production model. Massey composite updated via
CSV import; KenPom archive backfilled for all seasons 2010-2026.


Summary & Conclusions
-----------------------
~80% win accuracy was the ceiling for season-level statistical features alone.
Adding point-in-time rankings (consensus rank + PIT AdjEM) broke through to
81.7%, the first meaningful improvement beyond feature selection and model
architecture changes.

Features tested and found redundant:
- All 26 available KenPom + derived stats (only top 8 matter)
- Recent form (last 10 games)
- Roster composition (height, experience, continuity, bench)
- Point-in-time KenPom ratings alone (noisier than season-level)
- Momentum / 30-day rating change
- Separate offense + defense efficiency (vs combined margin)
- Rest days between games
- Opponent adjustment (playing up/down to competition)
- Home/away performance rank (team-specific home court advantage)
- 3pt shooting reliance (F3GRate, OppF3GRate)
- Travel distance between arenas
- Tournament seed matchups (redundant with efficiency ratings)
- Coach tenure / new coach indicator

Features that helped:
- PIT consensus rank (Massey composite): +10.1% importance, broke 80% ceiling
- PIT AdjEM (KenPom archive): +8.5% importance, complementary to season-level
- Contextual features (month, games played, conference): small but real gain

Model changes that helped:
- Switching from GBM to MLP: +0.2% win accuracy, 20x faster training
- Adding PIT features: +1.4% win accuracy (80.3% → 81.7%), MAE 7.59 → 7.39
- Fixing team ID duplicates: restored adj_efficiency_margin from 0.9%
  to 89.5% importance (data quality issue, not a modeling improvement)

Potential paths to further improvement:
- Vegas closing lines (market aggregates non-statistical information)
- Injury/availability data (no good historical source found yet)
- Fundamentally different approaches (player-level models, game simulation)
- More PIT features from other sources (e.g., individual system ranks)
